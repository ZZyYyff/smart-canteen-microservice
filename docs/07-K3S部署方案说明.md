# K3S 部署方案说明

## 1. K3S 安装

### 1.1 安装 K3S（单节点）

```bash
# 安装 K3S（默认安装最新稳定版）
curl -sfL https://get.k3s.io | sh -

# 验证安装
sudo k3s kubectl get nodes
```

### 1.2 配置 kubectl 别名（可选）

```bash
# 将 k3s 的 kubectl 配置写入用户目录
mkdir -p ~/.kube
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config
sudo chown $(id -u):$(id -g) ~/.kube/config

# 之后可直接使用 kubectl
kubectl get nodes
```

### 1.3 检查 K3S 状态

```bash
sudo systemctl status k3s    # systemd 管理的 K3S
kubectl get nodes             # 节点应为 Ready
kubectl get pods -A           # 系统 Pod 应为 Running
```

## 2. 检查节点状态

```bash
kubectl get nodes -o wide
# 预期输出:
# NAME   STATUS   ROLES                  AGE   VERSION
# host   Ready    control-plane,master   ...   v1.x.x+k3s

kubectl top nodes   # 查看资源使用（需安装 metrics-server）
```

## 3. 构建 Docker 镜像

每个微服务需要打包成 Docker 镜像并推送到 K3S 可访问的镜像仓库。

### 3.1 编写 Dockerfile（每个服务根目录）

以下为各模块的 Dockerfile 模板，以 `user-service` 为例：

```dockerfile
# user-service/Dockerfile
FROM eclipse-temurin:17-jre
WORKDIR /app
COPY target/user-service-1.0.0-SNAPSHOT.jar app.jar
EXPOSE 9001
ENTRYPOINT ["java", "-jar", "app.jar"]
```

> 同理创建 `menu-service/Dockerfile`、`order-service/Dockerfile`、`pickup-service/Dockerfile`、`gateway-service/Dockerfile`，替换对应端口号。

### 3.2 先打包 Jar

```bash
# 在项目根目录执行
mvn clean package -DskipTests
```

### 3.3 构建 Docker 镜像

```bash
# 为每个服务构建镜像
docker build -t smart-canteen/user-service:latest   -f user-service/Dockerfile .
docker build -t smart-canteen/menu-service:latest    -f menu-service/Dockerfile .
docker build -t smart-canteen/order-service:latest   -f order-service/Dockerfile .
docker build -t smart-canteen/pickup-service:latest  -f pickup-service/Dockerfile .
docker build -t smart-canteen/gateway-service:latest -f gateway-service/Dockerfile .

# 验证镜像
docker images | grep smart-canteen
```

### 3.4 导入镜像到 K3S

**方式一：使用 ctr 导入（推荐，适合单节点）**

```bash
# 导出镜像为 tar
docker save smart-canteen/user-service:latest \
  smart-canteen/menu-service:latest \
  smart-canteen/order-service:latest \
  smart-canteen/pickup-service:latest \
  smart-canteen/gateway-service:latest \
  -o smart-canteen-images.tar

# 导入到 k3s 内置的 containerd
sudo k3s ctr images import smart-canteen-images.tar

# 验证导入
sudo k3s ctr images ls | grep smart-canteen
```

**方式二：推送至 Docker Hub 或私有仓库**

```bash
# 标记并推送（以 Docker Hub 为例，替换 <username>）
docker tag smart-canteen/user-service:latest   <username>/smart-canteen-user-service:latest
docker push <username>/smart-canteen-user-service:latest
# ... 同理推送其他服务

# 然后更新 k8s/*.yaml 中的 image 字段为完整镜像地址
```

**方式三：使用 crictl 导入**

```bash
# K3S 使用 containerd 作为容器运行时
sudo k3s ctr images import smart-canteen-images.tar
```

## 4. 应用 YAML 文件

### 4.1 创建 Namespace 和基础设施

```bash
cd k8s

# 创建命名空间
kubectl apply -f namespace.yaml

# 部署 MySQL 初始化脚本 ConfigMap
kubectl apply -f configmap.yaml

# 部署基础设施（MySQL、Redis、Nacos）
kubectl apply -f mysql.yaml
kubectl apply -f redis.yaml
kubectl apply -f nacos.yaml

# 等待基础设施就绪
kubectl -n smart-canteen wait --for=condition=ready pod -l app=mysql --timeout=120s
kubectl -n smart-canteen wait --for=condition=ready pod -l app=redis --timeout=60s
kubectl -n smart-canteen wait --for=condition=ready pod -l app=nacos --timeout=120s
```

### 4.2 部署微服务

```bash
# 建议按依赖顺序部署
kubectl apply -f user-service.yaml
kubectl apply -f menu-service.yaml
kubectl apply -f order-service.yaml
kubectl apply -f pickup-service.yaml
kubectl apply -f gateway-service.yaml

# 一次性部署所有服务（可选）
kubectl apply -f ./
```

## 6. 访问网关

### 6.1 通过 NodePort 访问

Gateway 使用 NodePort 暴露，端口为 **30080**：

```bash
# 获取节点 IP
NODE_IP=$(kubectl get nodes -o jsonpath='{.items[0].status.addresses[0].address}')
echo "网关地址: http://${NODE_IP}:30080"

# 测试接口
curl http://${NODE_IP}:30080/api/users/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800000001","password":"123456"}'
```

### 6.2 通过 Port Forward 访问（开发调试）

```bash
# 将网关端口转发到本地
kubectl -n smart-canteen port-forward svc/gateway-service 8080:8080

# 另一终端测试
curl http://localhost:8080/api/menus/dishes \
  -H "Authorization: Bearer <token>"
```

### 6.3 通过 Ingress 访问（可选，需要配置 Ingress Controller）

K3S 默认安装了 Traefik Ingress Controller。创建 `ingress.yaml`：

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: smart-canteen
  namespace: smart-canteen
spec:
  rules:
    - http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: gateway-service
                port:
                  number: 8080
```

```bash
kubectl apply -f ingress.yaml
# 然后通过节点 IP（无需端口）直接访问: http://<NODE_IP>/
```

## 7. 验证服务注册

Nacos 控制台可通过 port-forward 访问：

```bash
kubectl -n smart-canteen port-forward svc/nacos 8848:8848
```

浏览器打开 `http://localhost:8848/nacos`，登录 `nacos/nacos`，在"服务管理 → 服务列表"中应看到：

- `gateway-service`
- `user-service`
- `menu-service`
- `order-service`
- `pickup-service`

## 8. 环境变量说明

| 环境变量 | 作用 | K3S 中的值 | 本地默认值 |
|----------|------|-----------|-----------|
| `MYSQL_HOST` | MySQL 主机地址 | `mysql`（K8S Service 名） | `localhost` |
| `MYSQL_PORT` | MySQL 端口 | `3306` | `3306` |
| `MYSQL_USER` | MySQL 用户名 | `root` | `root` |
| `MYSQL_PASSWORD` | MySQL 密码 | `root` | `root` |
| `REDIS_HOST` | Redis 主机地址 | `redis`（K8S Service 名） | `localhost` |
| `NACOS_SERVER_ADDR` | Nacos 注册中心地址 | `nacos:8848`（K8S Service 名） | `localhost:8848` |

> 每个服务的 `application.yml` 中已使用 `${ENV:default}` 占位符，K3S 环境中自动从环境变量取值，本地运行时回退到 `localhost`。

## 9. 常见问题排查

### 9.1 Pod 一直 Pending

```bash
kubectl -n smart-canteen describe pod <pod-name>
# 常见原因：节点资源不足、镜像拉取失败、PVC 未绑定
```

### 9.2 Pod CrashLoopBackOff

```bash
kubectl -n smart-canteen logs <pod-name> --tail=50
# 常见原因：数据库连接失败、Nacos 不可达、端口冲突
```

### 9.3 微服务无法注册到 Nacos

```bash
# 检查 Nacos 是否就绪
kubectl -n smart-canteen logs deployment/nacos --tail=20

# 检查微服务的 NACOS_SERVER_ADDR 配置
kubectl -n smart-canteen exec deployment/user-service -- env | grep NACOS

# 确认 DNS 解析正常
kubectl -n smart-canteen exec deployment/user-service -- nslookup nacos
```

### 9.4 数据库连接失败

```bash
# 检查 MySQL 是否就绪
kubectl -n smart-canteen get pods -l app=mysql

# 测试从微服务 Pod 连接 MySQL
kubectl -n smart-canteen exec deployment/user-service -- sh -c \
  "nc -zv mysql 3306"
```

### 9.5 镜像拉取失败

```bash
# 确认镜像已导入
sudo k3s ctr images ls | grep smart-canteen

# 如使用私有仓库，确认 imagePullSecrets 已配置
# 或使用 ctr 导入方式避免拉取问题
```

### 9.6 清理与重新部署

```bash
# 删除所有资源
kubectl delete namespace smart-canteen

# 重新部署
kubectl apply -f namespace.yaml
kubectl apply -f configmap.yaml
kubectl apply -f mysql.yaml
kubectl apply -f redis.yaml
kubectl apply -f nacos.yaml
kubectl apply -f user-service.yaml
kubectl apply -f menu-service.yaml
kubectl apply -f order-service.yaml
kubectl apply -f pickup-service.yaml
kubectl apply -f gateway-service.yaml
```
