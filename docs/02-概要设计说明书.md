# 概要设计说明书

## 1. 总体架构

本系统采用**微服务架构**，基于 Spring Cloud 技术栈实现。系统由 5 个微服务模块、1 个公共模块和 3 个基础设施组件组成。

```
                    ┌─────────────┐
                    │   客户端     │
                    └──────┬──────┘
                           │ HTTP / WS
                    ┌──────▼──────┐
                    │   Gateway   │  :8080
                    │  (网关层)    │
                    │ JWT + 限流   │
                    └──────┬──────┘
                           │
          ┌────────────────┼────────────────────┐
          │                │                    │
   ┌──────▼──────┐  ┌─────▼──────┐  ┌─────────▼──────┐
   │ user-service│  │menu-service│  │ order-service   │
   │   :9001     │  │   :9002    │  │    :9003        │
   └─────────────┘  └─────┬──────┘  └────────┬────────┘
                          │                  │
                          │ Feign            │ Feign
                          │                  │
                          │         ┌────────▼────────┐
                          │         │ pickup-service   │
                          │         │    :9004         │
                          │         │ WebSocket /ws    │
                          │         └─────────────────┘
                          │
          ┌───────────────┼─────────────────┐
          │               │                 │
   ┌──────▼──────┐  ┌─────▼──────┐  ┌──────▼──────┐
   │    MySQL    │  │   Redis    │  │    Nacos    │
   │   :3306     │  │   :6379    │  │   :8848     │
   └─────────────┘  └────────────┘  └─────────────┘
```

## 2. 微服务划分

| 服务 | 端口 | 职责 | 核心依赖 |
|------|------|------|---------|
| gateway-service | 8080 | 统一入口、JWT 校验、限流、路由转发 | Spring Cloud Gateway, Redis |
| user-service | 9001 | 用户注册、登录、JWT 颁发、个人信息 | MyBatis Plus, BCrypt |
| menu-service | 9002 | 菜品 CRUD、库存管理、每日菜单 | MyBatis Plus |
| order-service | 9003 | 订单创建、状态流转、Feign 调用 menu/pickup | OpenFeign, MyBatis Plus |
| pickup-service | 9004 | 窗口管理、排队叫号、取餐核销、WebSocket | OpenFeign, WebSocket |

## 3. 服务关系

### 3.1 服务调用关系

```
order-service ──Feign──► menu-service    (扣减/恢复库存)
order-service ──Feign──► pickup-service  (加入取餐队列)
pickup-service──Feign──► order-service   (完成订单)
```

### 3.2 网关路由

| 路由路径 | 目标服务 | 说明 |
|----------|---------|------|
| /api/users/** | lb://user-service | 用户相关接口 |
| /api/menus/** | lb://menu-service | 菜品与菜单接口 |
| /api/orders/** | lb://order-service | 订单相关接口 |
| /api/pickup/** | lb://pickup-service | 取餐相关接口 |
| /ws/pickup/** | lb://pickup-service | WebSocket 大屏推送 |

## 4. 数据库总体设计

### 4.1 数据库信息

- 数据库名称：`smart_canteen`
- 字符集：utf8mb4
- 引擎：InnoDB

### 4.2 核心 ER 关系

```
users ──1:N──► orders ──1:N──► order_items ◄──N:1── dishes
                       │
pickup_windows ──1:N──► pickup_queue ──N:1──► orders

daily_menus ──1:N──► daily_menu_items ◄──N:1── dishes
```

### 4.3 表清单

| 表名 | 所属模块 | 说明 |
|------|---------|------|
| users | user-service | 用户表 |
| dishes | menu-service | 菜品表 |
| daily_menus | menu-service | 每日菜单表 |
| daily_menu_items | menu-service | 每日菜单明细表 |
| orders | order-service | 订单表 |
| order_items | order-service | 订单明细表 |
| pickup_windows | pickup-service | 取餐窗口表 |
| pickup_queue | pickup-service | 取餐队列表 |

## 5. 主要接口设计

### 5.1 用户服务 `/api/users`

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| POST | /register | 用户注册 | 否 |
| POST | /login | 用户登录 | 否 |
| POST | /refresh-token | 刷新 Token | 否 |
| GET | /me | 查询当前用户 | 是 |
| PUT | /me | 修改当前用户 | 是 |

### 5.2 菜品与菜单服务 `/api/menus`

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| POST | /dishes | 新增菜品 | 是 |
| GET | /dishes | 查询菜品列表 | 是 |
| GET | /dishes/{id} | 查询菜品详情 | 是 |
| PUT | /dishes/{id} | 修改菜品 | 是 |
| DELETE | /dishes/{id} | 删除菜品 | 是 |
| PUT | /dishes/{id}/on-sale | 上架 | 是 |
| PUT | /dishes/{id}/off-sale | 下架 | 是 |
| PUT | /dishes/{id}/stock/deduct | 扣减库存 | Feign |
| PUT | /dishes/{id}/stock/restore | 恢复库存 | Feign |
| POST | /daily | 创建每日菜单 | 是 |
| GET | /today | 获取今日菜单 | 是 |

### 5.3 订单服务 `/api/orders`

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| POST | / | 创建订单 | 是 |
| GET | /{id} | 查询订单详情 | 是 |
| GET | /my | 查询我的订单 | 是 |
| GET | /merchant/pending | 商家查询待处理 | 是 |
| PUT | /{id}/cancel | 取消订单 | 是 |
| PUT | /{id}/accept | 商家接单 | 是 |
| PUT | /{id}/cooking | 开始制作 | 是 |
| PUT | /{id}/ready | 备餐完成 | 是 |
| PUT | /{id}/complete | 取餐完成 | Feign |

### 5.4 取餐服务 `/api/pickup`

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| POST | /windows | 新增窗口 | 是 |
| GET | /windows | 窗口列表 | 是 |
| PUT | /windows/{id}/enable | 启用窗口 | 是 |
| PUT | /windows/{id}/disable | 停用窗口 | 是 |
| POST | /queue | 加入取餐队列 | Feign |
| GET | /windows/{id}/queue | 查询队列 | 是 |
| POST | /windows/{id}/call-next | 叫下一号 | 是 |
| POST | /verify | 取餐核销 | 是 |

## 6. 技术选型说明

| 技术 | 版本 | 选型理由 |
|------|------|---------|
| Java | 17 | LTS 版本，生态成熟，Spring Boot 3.x 最低要求 |
| Spring Boot | 3.2.5 | 最新稳定版，自动配置简化开发 |
| Spring Cloud | 2023.0.3 | 微服务全家桶，与 Spring Boot 3.x 配套 |
| Spring Cloud Alibaba | 2023.0.1.0 | Nacos 服务发现，国产生态 |
| Spring Cloud Gateway | — | 基于 WebFlux 的高性能网关，比 Zuul 更适合异步场景 |
| OpenFeign | — | 声明式 HTTP 客户端，简化服务间调用 |
| Nacos | 2.3.2 | 同时支持服务发现和配置管理 |
| MyBatis Plus | 3.5.5 | 增强 MyBatis，简化 CRUD，支持 Lambda 查询 |
| MySQL | 8 | 开源关系型数据库，事务支持，成熟稳定 |
| Redis | 7 | 高性能缓存，支撑网关限流 |
| JWT (jjwt) | 0.12.5 | 无状态认证，适合微服务架构 |
| BCrypt | — | 密码单向哈希，Spring Security 内置 |
| Docker | — | 容器化部署，环境一致性 |
| K3S | — | 轻量级 Kubernetes，适合边缘和开发环境 |
