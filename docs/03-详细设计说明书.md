## 1. 用户服务详细设计

### 1.1 模块结构

```
│   ├── UserService.java                # 服务接口

    ├── UserVO.java                     # 用户视图（隐藏密码）
    └── LoginVO.java                    # 登录返回（token + user）
```

### 1.2 注册流程

```
客户端 → POST /api/users/register
  → UserController.register(dto)
    → UserServiceImpl.register(dto)
      1. 校验手机号是否已存在 → 存在则抛 BusinessException
      2. 校验学工号是否已存在 → 存在则抛 BusinessException
      3. BCryptPasswordEncoder.encode(password)
      4. 创建 User 实体（role=STUDENT, status=NORMAL）
      5. userMapper.insert(user)
      6. 返回 UserVO（隐藏密码字段）
    → Result.success("注册成功", vo)
```

### 1.3 登录流程

```
客户端 → POST /api/users/login
  → UserController.login(dto)
    → UserServiceImpl.login(dto)
      1. 通过手机号或学工号查找用户 → 不存在则抛 BusinessException
      2. BCryptPasswordEncoder.matches(password, user.password) → 不匹配则抛 BusinessException
      3. 检查 user.status == NORMAL → 禁用则抛 BusinessException
      4. JwtUtil.generateToken(userId, role) → accessToken (24h)
      5. JwtUtil.generateRefreshToken(userId, role) → refreshToken (7d)
      6. 返回 LoginVO{token, refreshToken, user}
    → Result.success("登录成功", vo)
```

### 1.4 Token 刷新

```
客户端 → POST /api/users/refresh-token {refreshToken: "xxx"}
  → UserController.refreshToken(body)
    → UserServiceImpl.refreshToken(refreshToken)
      1. JwtUtil.isRefreshTokenValid(refreshToken) → 无效则抛 BusinessException
      2. 提取 userId, role
      3. 生成新的 accessToken 和 refreshToken
      4. 返回 LoginVO
```

### 2.2 库存扣减 — 原子 SQL 设计

```sql
UPDATE dishes
SET stock = stock - #{quantity}, updated_at = NOW()
WHERE id = #{dishId} AND stock >= #{quantity}
```

返回值为 0 表示库存不足，1 表示扣减成功。Service 层判断返回值并抛 BusinessException(code=3002)。

### 2.3 库存恢复

```sql

WHERE id = #{dishId}
```

### 2.6 今日菜单查询算法

```
1. SELECT * FROM daily_menus WHERE menu_date = TODAY()
2. 遍历每个 DailyMenu:
   a. SELECT * FROM daily_menu_items WHERE menu_id = menu.id
   b. 收集所有 dish_id
   c. SELECT * FROM dishes WHERE id IN (dish_id 集合)
   d. 组装 DishVO 列表
3. 返回 DailyMenuVO 列表（含菜品详情）
```

### 3.2 下单流程（核心算法）

```
orderService.createOrder(userId, dto):
  1. 初始化 deductedItems = []
  2. for each item in dto.items:
     a. menuFeignClient.deductStock(item.dishId, StockRequest(quantity))
     b. if result.code != 200:
        → rollbackStock(deductedItems)  // 回滚已扣库存
        → throw BusinessException("库存不足")
     c. deductedItems.add(item)
  3. totalAmount = Σ(item.price × item.quantity)
  4. 创建 Order:
     - userId, windowId, totalAmount, status=CREATED
     - pickupNo = 100 + random(900)
     - pickupCode = String.format("%06d", random(1000000))
  5. orderMapper.insert(order)
  6. for each item: 创建 OrderItem 并插入
  7. return OrderVO.fromEntity(order, orderItems)
```

### 3.3 库存回滚

```
rollbackStock(deductedItems):
  for each item in deductedItems:
    try:
      menuFeignClient.restoreStock(item.dishId, StockRequest(quantity))
    catch: 记录日志（不回滚失败不阻塞主流程）
```

### 3.4 订单状态流转

```
                    ┌─────────┐
                    │ CREATED  │  已创建
                    └────┬─────┘
                         │ acceptOrder()
                    ┌────▼─────┐
                    │ ACCEPTED │  商家已接单
                    └────┬─────┘
                         │ cookingOrder()
                    ┌────▼─────┐
                    │ COOKING  │  制作中
                    └────┬─────┘
                         │ readyOrder()
                    ┌────▼──────┐
                    │ WAIT_PICKUP│ 待取餐（通知 pickup-service）
                    └────┬──────┘
                         │ completeOrder() ← pickup-service 调用
                    ┌────▼─────┐
                    │ COMPLETED│ 已完成
                    └──────────┘

   CREATED ──cancelOrder()──► CANCELLED
   ACCEPTED ──cancelOrder()──► CANCELLED
```

### 3.5 状态流转校验

伪代码：
```
transitionStatus(orderId, expectedCurrent, target):
  order = orderMapper.selectById(orderId)
  if order == null: throw NOT_FOUND
  if order.status != expectedCurrent: throw ORDER_STATUS_ERROR
  order.status = target
```

```
  1. 确认窗口存在
  2. SELECT * FROM pickup_queue
     WHERE window_id = windowId
       AND status = 'WAITING'
     ORDER BY queue_time ASC
     LIMIT 1
  3. 若为空 → throw NOT_FOUND("当前窗口没有排队记录")
  4. record.status = 'CALLED'
  5. record.call_time = now()
  6. queueMapper.updateById(record)
  7. 获取更新后的等待队列
  8. 构建 ScreenMessage:
     {type:"CALL", windowId, windowName, currentPickupNo, waitingQueue, timestamp, message}
  9. PickupScreenHandler.broadcast(JSON.stringify(msg))
  10. return PickupQueueVO
```

### 4.3 核销算法

```
verifyPickup(request):
  1. SELECT * FROM pickup_queue
     WHERE pickup_no = request.pickupNo
     ORDER BY queue_time DESC LIMIT 1
  2. 若为空 → throw NOT_FOUND("取餐记录不存在")
  3. 若 request.pickupCode != record.pickupCode → throw 5001("取餐码错误")
  4. 若 record.status == 'FINISHED' → throw 5002("该订单已取餐")
  5. 若 record.status != 'CALLED' → throw ORDER_STATUS_ERROR
  6. record.status = 'FINISHED'
  7. record.finish_time = now()
  8. queueMapper.updateById(record)
  9. orderFeignClient.completeOrder(record.orderId)  // 通知 order-service
  10. return PickupQueueVO
```

### 5.2 JWT 认证过滤器流程

```
JwtAuthFilter.filter(exchange, chain):
  1. path = request.getURI().getPath()
  2. if path in ["/api/users/login", "/api/users/register", "/api/users/refresh-token"]:
     → chain.filter(exchange)  // 放行
  3. authHeader = request.getHeader("Authorization")
  4. if authHeader == null or !startsWith("Bearer "):
     → 401 {"code":401,"message":"缺少认证 Token"}
  5. token = authHeader.substring(7)
  6. if JwtUtil.isTokenExpired(token):
     → 401 {"code":401,"message":"Token 已过期"}
  7. try:
     userId = JwtUtil.getUserId(token)
     role = JwtUtil.getRole(token)
     注入请求头:
       X-User-Id = userId
       X-User-Role = role
     chain.filter(exchange with mutated headers)
  8. catch → 401 {"code":401,"message":"Token 解析失败"}
```

### 5.3 限流配置

- 基础设施：RedisRateLimiter（基于 Redis 令牌桶算法）
- 默认速率：10 请求/秒，突发容量 20
- Key 解析：优先根据 X-User-Id 限流（用户级），未认证时根据 IP 限流
- 超出限制：HTTP 429
